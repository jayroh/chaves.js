Hi, I'm Joel. I am a developer living in Boston. I work at thoughtbot and I like keyboard shortcuts.

I'm not the only person who's ever thought "Wow, <INSERT WEBSITE/APP HERE> would be even more world-changing if it had some keyboard shortcuts." - am I right? If you're used to using sites like Gmail, Google reader, Github, etc then you're used to fast and efficient interactions just a key-press away.

If you don't use the keyboard shortcuts available to you then ... well.

There are handful of great JS libraries out there to help get your app sufficially shortcut'ed, but I found that there wasn't much in the way of a quick and flexible set of defaults with the ability to further configure. Convention over configuration. That's a thing, right?

So here's my shot at this. A small jQuery plugin called "Chaves.js" that leans on both jQuery and Thomas Fuch's Keymaster.js.

The element you call "chaves()" against will allow you to navigate its direct children, setting an 'active' class on the current child, with the "j" and "k" keys -- optionally with the up and down keys. "Enter" will click on the first anchor tag. The "/" key will focus on a search field, if it has a commonly named id, name or placeholder value. And, finally, "shift + /" -- or more commonly "?" -- will display a help dialog.

The help dialog is dynamically built from the registered key-bindings. If you would care to, instead, build that dialog with your own markup the plugin will hook into that instead. Just create an element with a class 'jquery-chaves-help' -- or, if you pass a different classname in the options it will use that classname.

If you'd like more details on how to customize things - visit the next demo.
