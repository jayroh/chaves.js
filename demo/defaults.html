Hi. My name's Joel. I am a developer living in Boston, I work at Thoughtbot and I like keyboard shortcuts.

I'm not the only person who's ever thought "Wow, <INSERT WEBSITE/APP HERE> sure would be even more awesome/fantastic/ hella rad/wicked pissah if it had some keyboard shortcuts." - am I right? If you're constantly using sites like Gmail, Google reader, Github, etc then you are pretty used to a full assortment of fast and efficient actions but a key-press or two away.

If you don't often use keyboard shortcuts available to you ... then I got nothing for you except a questioning glare tinged with disappointent. Seriously ... ?

There are handful of decent JS libraries out there to help you get your own app sufficially shortcut'ed, but I found that there wasn't much in the way of a quick and flexible set of defaults with the ability to configure as I see fit. Convention over configuration. That's a thing, right?

So here's my shot at this. A small jQuery plugin called "Chaves.js" that leans on both jQuery and Thomas Fuch's Keymaster.js. What does "chaves" mean? It's portuguese for "keys". I know - clever!

The element you call "chaves()" against will allow you to navigate its direct children, setting an 'active' class on the current child, with the "j" and "k" keys -- optionally with the up and down keys. "Enter" will click on the first anchor tag. The "/" key will focus on a search field, if it has a commonly named id, name or placeholder value. And, finally, "shift + /" -- or more commonly "?" -- will display a help dialog.

The help dialog is dynamically built from the registered key-bindings. If you would care to, instead, build that dialog with your own markup the plugin will hook into that instead. Just create an element with a class 'jquery-chaves-help' -- or, if you pass a different classname in the options it will use that classname.

If you'd like more details on how to customize things - visit the next demo.
